---
title: "Create Contact"
api: "POST https://api.clemta.com/contacts"
description: "Creates a new contact for a company"
---

## Authentication

This endpoint requires authentication using an API key. Include your API key in the request header:

```
X-API-Key: your_api_key_here
```

See [Authentication](/api-reference/authentication) for more details.

## Request Body

<ParamField body="type" type="string" required>
  Contact type. Must be either `customer` or `vendor`.
</ParamField>

<ParamField body="full_name" type="string" required>
  Full name of the contact.
</ParamField>

<ParamField body="email" type="string" required>
  Email address of the contact.
</ParamField>

<ParamField body="company_id" type="string" required>
  The ID of the company this contact belongs to.
</ParamField>

<ParamField body="status" type="string">
  Contact status (optional).
</ParamField>

<ParamField body="description" type="string">
  Description or notes about the contact.
</ParamField>

<ParamField body="cc_emails" type="array">
  Array of CC email addresses for the contact.
</ParamField>

<ParamField body="bcc_emails" type="array">
  Array of BCC email addresses for the contact.
</ParamField>

<ParamField body="billing" type="object">
  Billing information for the contact.
  
  <Expandable title="Billing Properties">
    <ParamField body="billing.email" type="string">
      Billing email address.
    </ParamField>
    
    <ParamField body="billing.phone" type="string">
      Phone number.
    </ParamField>
    
    <ParamField body="billing.phone_code" type="string">
      Phone country code (e.g., "+1").
    </ParamField>
    
    <ParamField body="billing.iso_code" type="string">
      Country ISO code (e.g., "US").
    </ParamField>
    
    <ParamField body="billing.country" type="string">
      Country name.
    </ParamField>
    
    <ParamField body="billing.address_1" type="string">
      Address line 1.
    </ParamField>
    
    <ParamField body="billing.address_2" type="string">
      Address line 2.
    </ParamField>
    
    <ParamField body="billing.postal_code" type="string">
      Postal code.
    </ParamField>
    
    <ParamField body="billing.district" type="string">
      District.
    </ParamField>
    
    <ParamField body="billing.state" type="string">
      State.
    </ParamField>
    
    <ParamField body="billing.province" type="string">
      Province.
    </ParamField>
    
    <ParamField body="billing.tax_id" type="string">
      Tax ID.
    </ParamField>
  </Expandable>
</ParamField>

## HTTP Response Codes

| Code | Description                                          |
| ---- | ---------------------------------------------------- |
| 200  | Contact created successfully                         |
| 400  | Invalid request data                                 |
| 401  | Unauthorized - Invalid or missing API key            |
| 404  | Company not found                                    |
| 500  | Internal server error                                |

## Error Response

When an error occurs, the API returns an error response:

```json
{
  "success": false,
  "message": "Invalid request data",
  "data": null,
  "error": {
    "code": "INVALID_REQUEST",
    "details": "Field 'email' is required"
  }
}
```
